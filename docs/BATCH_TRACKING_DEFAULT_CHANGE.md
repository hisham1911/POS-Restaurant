# 🔄 تغيير الخيار الافتراضي لتتبع دفعات المخزون

> **التاريخ:** 4 مايو 2026  
> **التغيير:** الخيار الافتراضي من **نشط** إلى **غير نشط**  
> **السبب:** معظم المنتجات لا تحتاج تتبع دفعات

---

## 📋 ملخص التغيير

### ❌ القيم القديمة:
```typescript
isBatchTracked: product?.isBatchTracked ?? true  // نشط افتراضياً ❌
```

### ✅ القيم الجديدة:
```typescript
isBatchTracked: product?.isBatchTracked ?? false  // غير نشط افتراضياً ✅
```

---

## 🎯 التأثير

### عند إضافة منتج جديد:

#### قبل التغيير:
```
المستخدم يفتح مودال إضافة منتج
└─ تتبع دفعات المخزون: ✅ نشط (افتراضياً)
   └─ يتم تتبع الدفعات والصلاحية (FEFO)
```

#### بعد التغيير:
```
المستخدم يفتح مودال إضافة منتج
└─ تتبع دفعات المخزون: ❌ غير نشط (افتراضياً)
   └─ لا يتم تتبع الدفعات (FIFO)
```

---

## 💡 السبب

### لماذا هذا التغيير؟

```
معظم المنتجات لا تحتاج تتبع دفعات:
├─ ملابس ❌ لا تحتاج
├─ إلكترونيات ❌ لا تحتاج
├─ أدوات منزلية ❌ لا تحتاج
├─ مواد غذائية ✅ تحتاج (قليلة)
└─ أدوية ✅ تحتاج (قليلة)

النتيجة:
- 80% من المنتجات لا تحتاج تتبع دفعات
- 20% فقط تحتاج (أدوية، مواد غذائية)
```

### الفائدة:
```
✅ المستخدم لا يحتاج تعطيل الخيار في كل مرة
✅ أسرع في إضافة المنتجات
✅ أقل احتمال للخطأ
✅ تجربة مستخدم أفضل
```

---

## 🔍 ما هو تتبع الدفعات؟

### تتبع الدفعات (Batch Tracking):
```
✅ نشط:
├─ كل دفعة لها رقم خاص
├─ كل دفعة لها تاريخ صلاحية
├─ كل دفعة لها سعر خاص
├─ البيع حسب FEFO (First Expired, First Out)
└─ مناسب للأدوية والمواد الغذائية

❌ غير نشط:
├─ لا يوجد رقم دفعة
├─ لا يوجد تاريخ صلاحية
├─ سعر واحد للمنتج
├─ البيع حسب FIFO (First In, First Out)
└─ مناسب لمعظم المنتجات
```

## 📊 المقارنة

| الميزة | قبل ❌ | بعد ✅ |
|--------|-------|-------|
| الخيار الافتراضي | نشط | غير نشط |
| عند إضافة منتج عادي | يحتاج تعطيل | جاهز مباشرة |
| عند إضافة دواء | جاهز مباشرة | يحتاج تفعيل |
| السرعة | أبطأ | أسرع |
| الأخطاء | أكثر | أقل |

### المودال عند الفتح:

#### قبل:
```

│ يتم تتبع الدفعات والصلاحية (FEFO) │
│                          [🟢 نشط]   │ ← نشط افتراضياً

```

#### بعد:
```
┌─────────────────────────────────────┐
│ تتبع دفعات المخزون                 │
│ لا يتم تتبع الدفعات (FIFO)         │
│                      [⚪ غير نشط]   │ ← غير نشط افتراضياً

```

## 📝 ملاحظات مهمة

### 1. **لا يؤثر على المنتجات الموجودة**
```
✅ المنتجات الموجودة تحتفظ بإعداداتها
✅ التغيير يؤثر على المنتجات الجديدة فقط
```

### 2. **يمكن التفعيل في أي وقت**
```
✅ المستخدم يمكنه تفعيل الخيار عند الحاجة
✅ مجرد نقرة واحدة على الزر
```

### 3. **الخدمات لا تتأثر**
```
✅ الخدمات (Services) لا تحتاج مخزون أصلاً
✅ الخيار يظهر للمنتجات المادية فقط
```

## 🎯 حالات الاستخدام

### حالة 1: إضافة منتج عادي (ملابس)
```
1. المستخدم يفتح مودال إضافة منتج
2. يملأ البيانات (اسم، سعر، إلخ)
3. تتبع الدفعات: ❌ غير نشط (افتراضياً) ✅
4. يضغط حفظ
5. المنتج يُضاف بدون تتبع دفعات ✅
```

### حالة 2: إضافة دواء
```
1. المستخدم يفتح مودال إضافة منتج
2. يملأ البيانات (اسم، سعر، إلخ)
3. تتبع الدفعات: ❌ غير نشط (افتراضياً)
4. المستخدم ينقر على الزر → ✅ نشط
5. يضغط حفظ
6. المنتج يُضاف مع تتبع دفعات ✅
```

## 🎓 التوصيات

### متى تفعّل تتبع الدفعات؟

```
✅ فعّل لـ:
├─ الأدوية (لها تاريخ صلاحية)
├─ المواد الغذائية (لها تاريخ صلاحية)
├─ المستحضرات التجميلية (لها تاريخ صلاحية)
└─ أي منتج له تاريخ انتهاء

❌ لا تفعّل لـ:
├─ الملابس
├─ الإلكترونيات
├─ الأدوات المنزلية
├─ الأثاث
└─ أي منتج بدون تاريخ انتهاء
```
