# 🔄 إعادة ترتيب أعمدة جدول العملاء

> **التاريخ:** 4 مايو 2026  
> **السبب:** المبلغ المستحق أهم من إجمالي المشتريات  
> **التحسين:** جعل المبلغ المستحق يظهر في شاشات أصغر

---

## 📋 ملخص التغيير

### ❌ الترتيب القديم:
```
1. الاسم
2. رقم الهاتف
3. العنوان (md+)
4. نقاط الولاء
5. إجمالي الطلبات (lg+)
6. إجمالي المشتريات (lg+)      ← كان هنا
7. المبلغ المستحق (xl+)          ← كان هنا
8. الإجراءات
```

### ✅ الترتيب الجديد:
```
1. الاسم
2. رقم الهاتف
3. العنوان (md+)
4. نقاط الولاء
5. إجمالي الطلبات (lg+)
6. المبلغ المستحق (lg+)          ← أصبح هنا ✅
7. إجمالي المشتريات (xl+)       ← أصبح هنا ✅
8. الإجراءات
```

---

## 🎯 الفوائد

### 1. **الأولوية للمعلومات المهمة**
```
✅ المبلغ المستحق أهم من إجمالي المشتريات
✅ يظهر أولاً في الجدول
✅ سهل الوصول إليه
```

### 2. **ظهور في شاشات أصغر**
```
قبل: المبلغ المستحق يظهر في xl+ فقط ❌
بعد: المبلغ المستحق يظهر في lg+ ✅

النتيجة:
- شاشات أكبر من 1024px → يظهر المبلغ المستحق ✅
- شاشات أصغر من 1024px → لا يظهر ❌
```

### 3. **إجمالي المشتريات في شاشات كبيرة فقط**
```
قبل: يظهر في lg+ ❌
بعد: يظهر في xl+ فقط ✅

النتيجة:
- شاشات أكبر من 1280px → يظهر إجمالي المشتريات ✅
- شاشات أصغر من 1280px → لا يظهر ❌
```

## 🎨 التصميم

### عمود المبلغ المستحق:
```
┌─────────────────────────────┐
│ المبلغ المستحق (كل الفروع) │
├─────────────────────────────┤
│ 1,500.00 ج.م               │ ← برتقالي bold
│ كل الفروع                  │ ← رمادي صغير
│ من 5,000.00 ج.م            │ ← حد الائتمان
└─────────────────────────────┘
```

**الميزات:**
- ✅ المبلغ برتقالي لافت للنظر
- ✅ توضيح "كل الفروع"
- ✅ عرض حد الائتمان (لو موجود)
- ✅ "—" لو مفيش مبلغ مستحق

## 🔧 التفاصيل التقنية

### Responsive Classes:

| العمود | Class | الظهور |
|--------|-------|--------|
| الاسم | - | دائماً |
| رقم الهاتف | - | دائماً |
| العنوان | `hidden md:table-cell` | md+ (768px+) |
| نقاط الولاء | - | دائماً |
| إجمالي الطلبات | `hidden lg:table-cell` | lg+ (1024px+) |
| **المبلغ المستحق** | `hidden lg:table-cell` | **lg+ (1024px+)** ✅ |
| إجمالي المشتريات | `hidden xl:table-cell` | xl+ (1280px+) |
| الإجراءات | - | دائماً |

## 📝 الكود

### Header (thead):
```tsx
<th className="... hidden lg:table-cell">
  إجمالي الطلبات
</th>
<th className="... hidden lg:table-cell">
  المبلغ المستحق (كل الفروع)
</th>
<th className="... hidden xl:table-cell">
  إجمالي المشتريات
</th>
```

### Body (tbody):
```tsx
<td className="... hidden lg:table-cell">
  {customer.totalOrders} طلب
</td>
<td className="... hidden lg:table-cell">
  {customer.totalDue > 0 ? (
    <div className="flex flex-col gap-1">
      <span className="font-semibold text-orange-600">
        {formatCurrency(customer.totalDue)}
      </span>
      <span className="text-xs text-gray-400">
        كل الفروع
      </span>
      {customer.creditLimit > 0 && (
        <span className="text-xs text-gray-500">
          من {formatCurrency(customer.creditLimit)}
        </span>
      )}
    </div>
  ) : (
    <span className="text-gray-400">—</span>
  )}
</td>
<td className="... hidden xl:table-cell">
  {formatCurrency(customer.totalSpent)}
</td>
```

## 💡 ملاحظات مهمة

### 1. **لماذا lg بدلاً من xl؟**
```
lg = 1024px+ (معظم اللابتوبات)
xl = 1280px+ (شاشات كبيرة فقط)

المبلغ المستحق مهم → يظهر في lg ✅
إجمالي المشتريات أقل أهمية → يظهر في xl فقط ✅
```

### 2. **الأولوية في التصميم**
```
الأهمية (من الأعلى للأقل):
1. الاسم + رقم الهاتف (دائماً)
2. نقاط الولاء (دائماً)
3. العنوان (md+)
4. إجمالي الطلبات (lg+)
5. المبلغ المستحق (lg+)      ← مهم جداً
6. إجمالي المشتريات (xl+)    ← أقل أهمية
```

### 3. **التوافق مع الموبايل**
```
✅ الموبايل يعرض الأعمدة الأساسية فقط
✅ التابلت يضيف العنوان
✅ اللابتوب يضيف الطلبات والمبلغ المستحق
✅ الديسكتوب يضيف إجمالي المشتريات
```
