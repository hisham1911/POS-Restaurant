## 👑 System Owner

### الهدف
إدارة النظام بالكامل والشركات المتعددة (Multi-tenant management)

### Scope
```
TenantId = NULL  → Global access
BranchId = NULL  → No specific branch
```

### Use Cases
- إنشاء شركات جديدة للعملاء
- تفعيل/تعطيل الشركات
- Restore من backup في حالات الطوارئ
- مراقبة صحة النظام (health checks)

### لا يستخدم لـ
- ❌ العمليات اليومية داخل الشركة
- ❌ إنشاء طلبات أو فواتير
- ❌ إدارة المخزون

### مثال واقعي
```
شركة KasserPro تبيع النظام لـ 10 مطاعم مختلفة.
SystemOwner هو موظف في KasserPro يدير حسابات المطاعم العشرة.
```

---

## 🔧 Admin (مدير الشركة)

### الهدف
إدارة شركة واحدة (Tenant) بكل فروعها

### Scope
```
TenantId = 1     → Specific tenant
BranchId = NULL  → All branches in tenant
```

### Use Cases
- إدارة الفروع والمستخدمين
- إعداد المنتجات والفئات
- مراجعة التقارير المالية
- إدارة المخزون والموردين
- إنشاء backup للبيانات

### لا يستخدم لـ
- ❌ إنشاء شركات جديدة
- ❌ الوصول لبيانات شركات أخرى
- ❌ إنشاء SystemOwner

### مثال واقعي
```
صاحب مطعم "البركة" لديه 3 فروع.
Admin هو المدير العام الذي يشرف على الفروع الثلاثة.
```

---

## 💼 Cashier (كاشير)

### الهدف
العمل اليومي في فرع واحد

### Scope
```
TenantId = 1  → Specific tenant
BranchId = 5  → Specific branch only
```

### Use Cases
- فتح/إغلاق الشفت
- إنشاء طلبات وفواتير
- استقبال المدفوعات
- إدارة العملاء (حسب الصلاحيات)

### الصلاحيات المخصصة
يتم تحديدها من قبل Admin:
- `OrdersCreate` - إنشاء طلبات
- `OrdersView` - عرض الطلبات
- `CustomersView` - عرض العملاء
- `ProductsView` - عرض المنتجات
- إلخ...

### لا يستخدم لـ
- ❌ إدارة المستخدمين
- ❌ تعديل الأسعار (إلا بصلاحية)
- ❌ عرض تقارير مالية شاملة
- ❌ الوصول لفروع أخرى

### مثال واقعي
```
أحمد يعمل كاشير في فرع "مدينة نصر" لمطعم "البركة".
يستطيع فقط العمل في فرعه ولا يرى بيانات الفروع الأخرى.
```

## 🔐 Role Hierarchy

```
SystemOwner (Global)
    │
    ├─► Tenant 1
    │   ├─► Admin (Tenant-wide)
    │   │   ├─► Branch A
    │   │   │   └─► Cashier 1, Cashier 2
    │   │   └─► Branch B
    │   │       └─► Cashier 3, Cashier 4
    │   
    └─► Tenant 2
        ├─► Admin (Tenant-wide)
        │   └─► Branch X
        │       └─► Cashier 5
```

## 🎯 متى تستخدم كل دور؟

### استخدم SystemOwner عندما:
- تدير شركة SaaS تبيع النظام لعملاء متعددين
- تحتاج إنشاء/تعطيل حسابات الشركات
- تحتاج restore من backup لشركة معينة

### استخدم Admin عندما:
- تدير شركة واحدة بفروع متعددة
- تحتاج إعداد المنتجات والأسعار
- تحتاج مراجعة التقارير المالية
- تحتاج إدارة الموظفين والصلاحيات

### استخدم Cashier عندما:
- تعمل في فرع واحد فقط
- مسؤول عن البيع اليومي
- لا تحتاج صلاحيات إدارية

## 🔄 Workflow مثالي

### إعداد نظام جديد

```
1. SystemOwner
   ↓ إنشاء Tenant "مطعم البركة"
   
2. Admin (تم إنشاؤه تلقائياً)
   ↓ تسجيل الدخول
   ↓ إنشاء 3 فروع
   ↓ إضافة المنتجات والفئات
   ↓ إنشاء 6 كاشيرين (2 لكل فرع)
   ↓ تحديد صلاحيات الكاشيرين
   
3. Cashiers
   ↓ تسجيل الدخول
   ↓ فتح الشفت
   ↓ البدء في البيع
```

- [ ] الإبلاغ عن أي مشاكل فوراً

## 🚨 أخطاء شائعة

### ❌ خطأ 1: استخدام SystemOwner للعمل اليومي
```
SystemOwner ليس للعمليات اليومية!
استخدم Admin أو Cashier بدلاً منه.
```

### ❌ خطأ 2: إعطاء Cashier صلاحيات Admin
```
لا تعطي الكاشير صلاحيات إدارية.
استخدم نظام الصلاحيات المخصص.
```

### ❌ خطأ 3: Admin يحاول إنشاء SystemOwner
```
Admin لا يستطيع إنشاء SystemOwner.
هذا محمي أمنياً (privilege escalation prevention).
```

### ❌ خطأ 4: Cashier يصل لفروع أخرى
```
Cashier مقيد بفرع واحد فقط.
إذا احتاج العمل في فرع آخر، أنشئ حساب جديد.
```
