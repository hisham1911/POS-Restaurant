---

## ❓ ما هو زر "مطابقة وإغلاق الشيفت"؟

### التعريف البسيط

```
زر "مطابقة وإغلاق الشيفت" = التأكد من أن الفلوس في الدرج = الفلوس في النظام
```

```
في نهاية اليوم:
├─ النظام يقول: عندك 5000 ج.م في الدرج
├─ لكن الفلوس الفعلية في الدرج: ؟؟؟
```

## 🔄 كيف يعمل

### الخطوات بالتفصيل:

```
1. في نهاية الشيفت (نهاية اليوم)
   └─ الكاشير يضغط "مطابقة وإغلاق الشيفت"

2. النظام يعرض:
   ┌─────────────────────────────────┐
   │ الرصيد المتوقع (حسابي):        │
   │ 5,000.00 ج.م                   │ ← من النظام
   └─────────────────────────────────┘

3. الكاشير يعد الفلوس في الدرج:
   ├─ يعد كل الفئات (200، 100، 50، 20، 10، 5، 1)
   └─ الإجمالي: 4,950 ج.م

4. الكاشير يدخل المبلغ الفعلي:
   ┌─────────────────────────────────┐
   │ المبلغ الفعلي في الدرج:        │
   │ [4,950.00]                     │ ← يكتبه الكاشير
   └─────────────────────────────────┘

5. النظام يحسب الفرق:
   ┌─────────────────────────────────┐
   │ الفرق:                         │
   │ -50.00 ج.م (نقص)              │ ← تلقائياً
   └─────────────────────────────────┘

6. الكاشير يكتب ملاحظات (اختياري):
   ┌─────────────────────────────────┐
   │ ملاحظات:                       │
   │ [فكة ناقصة 50 جنيه]           │
   └─────────────────────────────────┘

7. يضغط "تأكيد المطابقة"
   └─ الشيفت يقفل تلقائياً ✅
```

## 💡 الحالات المختلفة

### الحالة 1: الفلوس متطابقة تماماً ✅

```
الرصيد المتوقع: 5,000.00 ج.م
المبلغ الفعلي:   5,000.00 ج.م
الفرق:           0.00 ج.م

النتيجة:
┌─────────────────────────────────┐
│ ✅ ممتاز! الحسابات متطابقة     │
│ الفرق: 0.00 ج.م               │
└─────────────────────────────────┘

المعنى:
├─ كل شيء صحيح
├─ لا مشاكل
└─ الكاشير شغله ممتاز
```

### الحالة 2: فلوس زيادة 💰

```
الرصيد المتوقع: 5,000.00 ج.م
المبلغ الفعلي:   5,100.00 ج.م
الفرق:           +100.00 ج.م (زيادة)

النتيجة:
┌─────────────────────────────────┐
│ ⚠️ تنبيه: فلوس زيادة           │
│ الفرق: +100.00 ج.م            │
└─────────────────────────────────┘

الأسباب المحتملة:
├─ خطأ في الحساب (الكاشير عد غلط)
├─ خطأ في إدخال فاتورة (سعر أقل من الفعلي)
├─ عميل دفع زيادة ونسي الباقي
└─ خطأ في الفكة (أعطى فكة أقل)
```

### الحالة 3: فلوس ناقصة ⚠️

```
الرصيد المتوقع: 5,000.00 ج.م
المبلغ الفعلي:   4,950.00 ج.م
الفرق:           -50.00 ج.م (نقص)

النتيجة:
┌─────────────────────────────────┐
│ ⚠️ تنبيه: فلوس ناقصة            │
│ الفرق: -50.00 ج.م             │
└─────────────────────────────────┘

الأسباب المحتملة:
├─ خطأ في الحساب (الكاشير عد غلط)
├─ خطأ في إدخال فاتورة (سعر أعلى من الفعلي)
├─ فكة زيادة للعميل (أعطى فكة أكثر)
├─ سرقة (نادر لكن ممكن)
└─ مصاريف صغيرة لم تُسجل
```

## 🎬 سيناريو عملي كامل

### يوم عمل في الصيدلية:

```
الصباح (9:00 ص):
├─ الكاشير يفتح الشيفت
├─ رصيد البداية: 1,000 ج.م (فكة)
└─ يبدأ البيع

طوال اليوم:
├─ مبيعات نقدية: 8,500 ج.م
├─ مبيعات بطاقة: 2,000 ج.م (لا تدخل الدرج)
├─ مصروفات: -500 ج.م (شراء فكة)
└─ الرصيد المتوقع = 1,000 + 8,500 - 500 = 9,000 ج.م

المساء (6:00 م):
├─ الكاشير يضغط "مطابقة وإغلاق الشيفت"
├─ النظام يعرض: الرصيد المتوقع = 9,000 ج.م
├─ الكاشير يعد الفلوس في الدرج
├─ الفلوس الفعلية: 8,980 ج.م
├─ الفرق: -20 ج.م (نقص)
├─ الكاشير يكتب: "فكة ناقصة 20 جنيه"
└─ يضغط "تأكيد المطابقة" ✅

النتيجة:
├─ الشيفت يقفل تلقائياً
├─ التقرير يحفظ في النظام
├─ المدير يراجع الفرق (-20 ج.م)
└─ يتخذ الإجراء المناسب
```

## 🎯 الفوائد الرئيسية

### 1. الشفافية والمحاسبة

```
✅ كل شيء موثق:
├─ الرصيد المتوقع
├─ الرصيد الفعلي
├─ الفرق
├─ الملاحظات
├─ التاريخ والوقت
└─ اسم الكاشير

النتيجة:
├─ لا خلافات
├─ لا شكوك
└─ كل شيء واضح
```

### 2. اكتشاف الأخطاء

```
✅ اكتشاف الأخطاء مبكراً:
├─ خطأ في الحساب
├─ خطأ في إدخال الفواتير
├─ خطأ في الفكة
└─ مشاكل في النظام

النتيجة:
├─ حل المشاكل بسرعة
└─ تحسين الأداء
```

### 3. منع السرقة

```
✅ رادع للسرقة:
├─ الكاشير يعرف إنه هيتحاسب
├─ كل فلس محسوب
└─ الفروقات واضحة

النتيجة:
├─ أمان أكثر
└─ ثقة أكبر
```

### 4. تقارير دقيقة

```
✅ تقارير مالية دقيقة:
├─ إجمالي المبيعات النقدية
├─ الفروقات اليومية
├─ أداء كل كاشير
└─ الاتجاهات (Trends)

النتيجة:
├─ قرارات أفضل
└─ إدارة أحسن
```

## 🐛 المشاكل الشائعة وحلولها

### المشكلة 1: الفرق كبير جداً

```
مثال: الفرق = -500 ج.م

الأسباب المحتملة:
├─ خطأ في العد (الأكثر شيوعاً)
├─ فاتورة كبيرة لم تُسجل
├─ مصروف كبير لم يُسجل
└─ مشكلة في النظام (نادر)

الحل:
1. أعد عد الفلوس مرة أخرى
2. راجع الفواتير الكبيرة
3. راجع المصروفات
4. راجع سجل الحركات
5. اتصل بالدعم الفني (لو لازم)
```

### المشكلة 2: الزر غير ظاهر

```
السبب:
├─ مفيش شيفت مفتوح
└─ أو مفيش صلاحية

الحل:
├─ افتح شيفت أولاً
└─ أو اطلب الصلاحية من المدير
```

### المشكلة 3: لا يمكن إغلاق الشيفت

```
السبب:
├─ في طلبات مفتوحة (Draft)
└─ أو في مشكلة في البيانات

الحل:
├─ أتمم أو ألغي الطلبات المفتوحة
└─ ثم حاول مرة أخرى
```

## 💡 نصائح مهمة

### 1. عد الفلوس بدقة

```
✅ الصح:
├─ اعزل كل فئة لوحدها
├─ عد كل فئة مرتين
├─ استخدم آلة عد (لو متاحة)
└─ اكتب الأرقام على ورقة

❌ الغلط:
├─ عد سريع بدون تركيز
└─ عد مرة واحدة فقط
```

### 2. سجل الملاحظات

```
✅ الصح:
├─ اكتب سبب الفرق (لو عارفه)
├─ اكتب أي ملاحظات مهمة
└─ كن صريح وواضح

مثال:
├─ "فكة ناقصة 20 جنيه"
├─ "عميل دفع 100 زيادة ونسي الباقي"
└─ "خطأ في عد الفئات الصغيرة"
```

### 3. طابق يومياً

```
✅ الصح:
├─ طابق في نهاية كل شيفت
├─ لا تترك الشيفت مفتوح أيام
└─ كلما أسرع، كلما أفضل

❌ الغلط:
├─ ترك الشيفت مفتوح أسبوع
└─ المطابقة مرة في الشهر
```

### 4. راجع التقارير

```
✅ الصح:
├─ المدير يراجع تقارير المطابقة يومياً
├─ يتابع الفروقات
├─ يحلل الأسباب
└─ يتخذ إجراءات تصحيحية

النتيجة:
├─ تحسين مستمر
└─ أخطاء أقل
```

## 🎯 الخلاصة

### زر "مطابقة وإغلاق الشيفت" يساعدك في:

```
1. التأكد من دقة الحسابات
   └─ الفلوس في الدرج = الفلوس في النظام

2. اكتشاف الأخطاء مبكراً
   └─ قبل ما تكبر المشكلة

3. المحاسبة والشفافية
   └─ كل شيء موثق ومسجل

4. منع السرقة
   └─ رادع قوي

5. تقارير دقيقة
   └─ قرارات أفضل

6. إغلاق الشيفت تلقائياً
   └─ جاهز لشيفت جديد
```

### متى تستخدمه؟

```
✅ في نهاية كل شيفت (يومياً)
✅ قبل تسليم الخزينة لكاشير آخر
✅ في نهاية اليوم
✅ عند تغيير الوردية
```

### لماذا هو مهم؟

```
بدون المطابقة:
❌ لا تعرف لو في فلوس ناقصة
❌ لا تعرف لو في أخطاء
❌ لا تعرف أداء الكاشير
❌ لا تقارير دقيقة

مع المطابقة:
✅ كل شيء واضح
✅ كل شيء موثق
✅ كل شيء دقيق
✅ إدارة احترافية
```
