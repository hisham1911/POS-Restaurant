# 📊 دليل RealisticDataSeeder - مولد البيانات الواقعية

## 🎯 نظرة عامة

`RealisticDataSeeder` هو نظام متقدم لتوليد بيانات واقعية شاملة لنظام KasserPro POS. يغطي **120 يوم كاملة** من البيانات التشغيلية مع سيناريوهات حقيقية ومتنوعة.

---

## ✨ المميزات الرئيسية

### 🔍 الشمولية الكاملة
- ✅ **كل جدول في قاعدة البيانات** له بيانات - لا يوجد جدول فارغ
- ✅ **120 يوم** من البيانات التاريخية (4 شهور كاملة)
- ✅ **علاقات صحيحة** بين كل الكيانات (Foreign Keys)
- ✅ **ترتيب زمني منطقي** - التواريخ موزعة بشكل واقعي

### 💼 ذكاء الأعمال
- 📈 **10-25 طلب يومياً** حسب نوع اليوم (عادي/عطلة)
- 💰 **20% من الطلبات** بها ديون (AmountPaid < Total)
- 📦 **فواتير شراء** من موردين متعددين
- 💸 **مصروفات شهرية** ثابتة + مصروفات عشوائية
- ↩️ **مرتجعات** كاملة وجزئية (5% من الطلبات)

### 🎲 التنوع والواقعية
- 👥 **20 عميل** (VIP، منتظمين، جدد، جملة)
- 🏪 **5 موردين** مع شروط دفع مختلفة
- 📦 **33 منتج** متنوع (بقالة، مشروبات، منظفات)
- 💳 **طرق دفع متنوعة**: Cash (50%)، Card (35%)، Fawry (10%)، Bank Transfer (5%)
- 📊 **حالات الطلبات**: Completed (85%)، Cancelled (10%)، Refunded (5%)

```

   ├─ 5 Cashiers
   ├─ 5 Suppliers
   ├─ 20 Customers
   └─ 9 Expense Categories

   └─ 33 Products (بقالة، مشروبات، منظفات)

   └─ SupplierProducts (8-12 منتج لكل مورد)

4️⃣ إنشاء فواتير الشراء (120 يوم)
   ├─ PurchaseInvoices (1-2 فاتورة أسبوعياً)
   ├─ PurchaseInvoiceItems
   └─ PurchaseInvoicePayments (60% مدفوع، 30% جزئي، 10% آجل)

5️⃣ إنشاء الورديات والطلبات (120 يوم)
   ├─ Shifts (2 وردية يومياً × 120 يوم = 240 وردية)
   ├─ Orders (10-25 طلب/يوم = ~1800 طلب)
   ├─ OrderItems
   ├─ Payments
   └─ StockMovements (حركة مخزون لكل بيع)

6️⃣ إنشاء المصروفات
   ├─ رواتب شهرية (18,000 ج.م)
   ├─ إيجار شهري (12,000 ج.م)
   ├─ فواتير (كهرباء، مياه)
   └─ مصروفات عشوائية (صيانة، مواصلات، تسويق)

7️⃣ إنشاء معاملات الخزينة
   └─ CashRegisterTransactions (إيداع/سحب)

8️⃣ إنشاء سدادات الديون
   └─ DebtPayments (50% من الديون يتم سدادها)

9️⃣ إنشاء المرتجعات
   ├─ RefundLogs (5% من الطلبات)
   └─ StockMovements (إرجاع المخزون)

🔟 تحديث المخزون
   └─ BranchInventories (حساب المخزون النهائي)
```

## 🚀 كيفية الاستخدام

### الطريقة الأولى: تشغيل تلقائي (موصى به)

الـ Seeder يعمل تلقائياً عند تشغيل التطبيق:

```bash
cd backend/KasserPro.API
dotnet run
```

سيتحقق النظام من وجود البيانات:
- ✅ إذا كان عدد الطلبات > 500 → يتخطى التحميل
- 🔄 إذا كان عدد الطلبات < 500 → يبدأ التحميل

### الطريقة الثانية: تشغيل يدوي

إذا أردت إعادة تحميل البيانات:

1. **احذف قاعدة البيانات**:
```bash
rm backend/KasserPro.API/kasserpro.db
```

2. **شغل التطبيق**:
```bash
cd backend/KasserPro.API
dotnet run
```

## 🛡️ الأمان والأداء

### ✅ الأمان
- 🔒 **Fixed Seed** (2026) - نتائج متسقة
- 🔒 **لا يمسح System Owner** - محمي
- 🔒 **يمسح فقط بيانات السوبر ماركت** - آمن

### ⚡ الأداء
- 🚀 **Batch Insert** - استخدام `AddRange()` بدلاً من `Add()`
- 🚀 **Single SaveChanges** - حفظ مجمع لكل مجموعة
- 🚀 **No N+1 Queries** - استخدام `Include()` بذكاء
- 🚀 **وقت التنفيذ**: ~30-60 ثانية لـ 120 يوم

---

## 🐛 استكشاف الأخطاء

### المشكلة: "لم يتم العثور على سوبر ماركت"
**الحل**: شغل `MultiTenantSeeder` أولاً:
```csharp
await MultiTenantSeeder.SeedAsync(context);
await RealisticDataSeeder.SeedAsync(context);
```

### المشكلة: "البيانات موجودة مسبقاً"
**الحل**: احذف قاعدة البيانات وأعد التشغيل:
```bash
rm backend/KasserPro.API/kasserpro.db
dotnet run
```

### المشكلة: "Foreign Key Constraint"
**الحل**: تأكد من ترتيب الحذف في `ClearExistingSupermarketDataAsync()`

## 📝 ملاحظات مهمة

1. ⚠️ **لا يمسح System Owner** - محمي من الحذف
2. ⚠️ **يمسح فقط بيانات السوبر ماركت** - المحلات الأخرى آمنة
3. ⚠️ **يتحقق من وجود البيانات** - لا يكرر التحميل
4. ⚠️ **يحترم Foreign Keys** - ترتيب الحذف والإضافة صحيح
5. ⚠️ **Egypt Timezone** - كل التواريخ بتوقيت مصر (UTC+2)
