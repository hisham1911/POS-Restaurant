## ما هو الباركود؟

### التعريف
**الباركود** هو رمز بصري (خطوط عمودية) يمثل رقم فريد لكل منتج.

```
مثال باركود:
|||  |  ||  |||  |  ||  |||
6281000123456
```

### الفرق بين الباركود والـ SKU

| الخاصية | Barcode (الباركود) | SKU |
|---------|-------------------|-----|
| **من يضعه؟** | الشركة المصنعة (عالمي) | أنت (داخلي) |
| **الشكل** | أرقام فقط (13 رقم عادةً) | حروف + أرقام |
| **المعيار** | UPC/EAN (عالمي) | حسب نظامك |
| **الاستخدام** | مسح ضوئي للبيع | إدارة داخلية |
| **المثال** | `6281000123456` | `TSH-RED-M` |

## كيف يعمل الباركود في نظام POS؟

### الخطوات:

```
1. المنتج يجي من المورد بباركود مطبوع عليه
   └─ مثال: شامبو بانتين → باركود: 8001090313690

2. تضيف المنتج في النظام وتكتب رقم الباركود
   └─ في KasserPro: حقل "الباركود"

3. في نقطة البيع:
   ├─ الكاشير يمسح الباركود بالماسح الضوئي
   ├─ النظام يبحث عن المنتج بالباركود
   ├─ يضيف المنتج للسلة تلقائياً
   └─ يخصم من المخزون

4. الفوائد:
   ├─ سرعة فائقة (ثانية واحدة)
   ├─ لا أخطاء إملائية
   ├─ لا حاجة للبحث اليدوي
   └─ تحديث المخزون تلقائياً
```

## القاعدة الذهبية ⭐

### **كل مقاس/لون/حجم = باركود مختلف!**

> **المصدر:** [GS1 (المنظمة العالمية للباركود)](https://gs1upc.net/barcodes-for-garments)

#### لماذا؟

```
السبب 1: تتبع المخزون بدقة
├─ تيشيرت أحمر M → باركود: 6281000123456
├─ تيشيرت أحمر L → باركود: 6281000123457
└─ النظام يعرف أي مقاس نفد

السبب 2: معرفة الأكثر مبيعاً
├─ مقاس M → بيع 50 قطعة
├─ مقاس L → بيع 30 قطعة
└─ تعرف تطلب مقاس M أكتر

السبب 3: السرعة في البيع
├─ الكاشير يمسح الباركود
└─ المنتج الصح يضاف للسلة مباشرة
```

## أمثلة عملية

### مثال 1: محل ملابس 👕

```
المنتج: تيشيرت أحمر

في النظام:
├─ منتج 1: تيشيرت أحمر - مقاس S
│   └─ الباركود: 6281000123450
│
├─ منتج 2: تيشيرت أحمر - مقاس M
│   └─ الباركود: 6281000123451
│
├─ منتج 3: تيشيرت أحمر - مقاس L
│   └─ الباركود: 6281000123452
│
└─ منتج 4: تيشيرت أحمر - مقاس XL
    └─ الباركود: 6281000123453

في نقطة البيع:
├─ العميل: "عايز مقاس M"
├─ الكاشير: يمسح باركود 6281000123451
└─ النظام: يضيف "تيشيرت أحمر M" للسلة ✅
```

### مثال 2: سوبر ماركت 🛒

```
المنتج: شامبو بانتين

في النظام:
├─ منتج 1: شامبو بانتين 200 مل
│   └─ الباركود: 8001090313690
│
├─ منتج 2: شامبو بانتين 400 مل
│   └─ الباركود: 8001090313691
│
└─ منتج 3: شامبو بانتين 1 لتر
    └─ الباركود: 8001090313692

في نقطة البيع:
├─ الكاشير: يمسح الباركود على العبوة
└─ النظام: يضيف الحجم الصح تلقائياً ✅
```

### مثال 3: محل أحذية 👟

```
المنتج: حذاء رياضي Nike

في النظام:
├─ منتج 1: Nike Air - مقاس 40 - أسود
│   └─ الباركود: 0193655123450
│
├─ منتج 2: Nike Air - مقاس 40 - أبيض
│   └─ الباركود: 0193655123451
│
├─ منتج 3: Nike Air - مقاس 42 - أسود
│   └─ الباركود: 0193655123452
│
└─ منتج 4: Nike Air - مقاس 42 - أبيض
    └─ الباركود: 0193655123453

كل مقاس + لون = باركود مختلف!
```

## كيفية إضافة الباركود في KasserPro

### الطريقة الحالية (الصحيحة):

#### الخطوة 1: أضف كل مقاس كمنتج منفصل

```
المنتج 1:
├─ الاسم: تيشيرت أحمر - مقاس S
├─ الباركود: 6281000123450  ← من الملصق على المنتج
├─ السعر: 100 ج.م
└─ الكمية: 10

المنتج 2:
├─ الاسم: تيشيرت أحمر - مقاس M
├─ الباركود: 6281000123451  ← من الملصق على المنتج
├─ السعر: 100 ج.م
└─ الكمية: 15

المنتج 3:
├─ الاسم: تيشيرت أحمر - مقاس L
├─ الباركود: 6281000123452  ← من الملصق على المنتج
├─ السعر: 100 ج.م
└─ الكمية: 20
```

#### الخطوة 2: في نقطة البيع

```
1. الكاشير يمسح الباركود بالماسح الضوئي
   └─ مثال: يمسح 6281000123451

2. النظام يبحث عن المنتج بالباركود
   └─ يجد: "تيشيرت أحمر - مقاس M"

3. يضيف المنتج للسلة تلقائياً
   └─ السعر: 100 ج.م
   └─ الكمية: 1

4. يخصم من المخزون
   └─ الكمية المتاحة: 15 → 14
```

## أنواع الباركود

### 1. UPC (Universal Product Code)
```
الشكل: 12 رقم
المثال: 012345678905
الاستخدام: أمريكا وكندا
```

### 2. EAN-13 (European Article Number)
```
الشكل: 13 رقم
المثال: 6281000123456
الاستخدام: مصر وأوروبا والعالم
الأكثر شيوعاً في مصر ✅
```

### 3. EAN-8
```
الشكل: 8 أرقام
المثال: 12345670
الاستخدام: المنتجات الصغيرة
```

### 4. Code 128
```
الشكل: حروف + أرقام
المثال: ABC-123-XYZ
الاستخدام: الشحن والمخازن
```

## الأسئلة الشائعة

### س1: لو المنتج مجاش من المورد بباركود؟

**الجواب:**
```
الحل 1: اطبع باركود خاص بيك
├─ اشتري طابعة باركود (1000-3000 ج.م)
├─ اطبع ملصقات باركود
└─ الصقها على المنتجات

الحل 2: استخدم البحث اليدوي
├─ الكاشير يكتب اسم المنتج
└─ أبطأ لكن يشتغل
```

### س2: هل لازم أشتري ماسح باركود؟

**الجواب:**
```
لا، لكن مفيد جداً!

بدون ماسح:
├─ الكاشير يكتب الباركود يدوياً
└─ أبطأ + احتمال أخطاء

مع ماسح (300-1000 ج.م):
├─ مسح سريع (ثانية واحدة)
├─ لا أخطاء
└─ تجربة احترافية
```

### س3: الباركود بيتكرر؟

**الجواب:**
```
لا! كل باركود فريد عالمياً

مثال:
├─ شامبو بانتين 500 مل → 8001090313690
└─ هذا الرقم لا يتكرر في العالم كله!

لكن في نظامك:
├─ يمكنك استخدام نفس الباركود لمنتجات مختلفة
└─ لكن مش recommended!
```

### س4: كل قطعة لها باركود مختلف؟

**الجواب:**
```
لا! نفس المنتج = نفس الباركود

✅ الصح:
├─ 10 قطع تيشيرت أحمر M
└─ كلهم باركود: 6281000123451 (نفس الباركود!)

❌ الغلط:
├─ كل قطعة لها باركود مختلف
└─ ده مش عملي ومش ضروري

لكن:
مقاس مختلف = باركود مختلف:
├─ تيشيرت أحمر M → 6281000123451
└─ تيشيرت أحمر L → 6281000123452

💡 الباركود بيعرّف نوع المنتج، مش القطعة الفردية
```

### س4.1: طيب لو عندي 10 علب دواء من نفس النوع؟

**الجواب:**
```
✅ كلهم نفس الباركود!

مثال:
10 علب Panadol 500mg
├─ علبة 1: باركود 6281234567890
├─ علبة 2: باركود 6281234567890
├─ علبة 3: باركود 6281234567890
├─ ... (كلهم نفس الباركود)
└─ علبة 10: باركود 6281234567890

في نقطة البيع:
├─ تمسح الباركود مرة واحدة
└─ تدخل الكمية: 10

💡 الباركود بيعرّف المنتج، مش العلبة الفردية
```

**ملاحظة مهمة للأدوية:**
```
لو عندك دفعتين مختلفتين (Batch):
├─ دفعة 1: 5 علب (Batch: LOT-A, Expiry: 2026-12)
├─ دفعة 2: 5 علب (Batch: LOT-B, Expiry: 2027-03)
└─ نفس الباركود، لكن باتش مختلف

في KasserPro:
├─ المنتج واحد (نفس الباركود)
└─ باتشين مختلفين (تواريخ انتهاء مختلفة)

📖 للتفاصيل الكاملة عن الأدوية:
راجع: docs/PHARMACEUTICAL_BARCODE_EXPLAINED.md
```

### س5: إزاي أعرف الباركود الصح؟

**الجواب:**
```
الباركود موجود على:
├─ الملصق على المنتج
├─ العبوة
└─ الكرتونة

مثال:
┌─────────────────────┐
│ شامبو بانتين       │
│ 500 مل             │
│                     │
│ |||  |  ||  |||    │
│ 8001090313690      │ ← هنا!
└─────────────────────┘
```

## الخلاصة النهائية 🎬

### القواعد الأساسية:

```
1. كل مقاس/لون/حجم = منتج منفصل ✅
2. كل منتج منفصل = باركود مختلف ✅
3. الباركود يجي من المورد (مطبوع على المنتج) ✅
4. تكتب الباركود في حقل "الباركود" في النظام ✅
5. في نقطة البيع: مسح الباركود = إضافة للسلة ✅
```

### في KasserPro حالياً:

```
✅ النظام يدعم الباركود بالكامل
✅ البحث بالباركود في POS شغال
✅ كل منتج له حقل باركود
✅ يمكن استخدام ماسح باركود USB

❌ مفيش Product Variants (كل مقاس = منتج منفصل)
```
