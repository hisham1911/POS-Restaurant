# إصلاح عرض سعر الباتش في نموذج تعديل المنتج

## المشكلة السابقة

عند تعديل منتج له دفعات (Batches):
- حقل "سعر البيع" كان يعرض **السعر الأساسي** (`price`) = 80 ج.م
- لكن السعر الفعلي المستخدم في البيع هو **سعر الباتش** (`suggestedPrice`) = 18 ج.م
- هذا كان يسبب ارتباك للمستخدم

## الحل المطبق

### التغيير الرئيسي
**الملف:** `frontend/src/components/products/ProductFormModal.tsx`

```typescript
// قبل التعديل
price: product?.price || 0,  // كان يعرض 80 ج.م

// بعد التعديل
price: product?.suggestedPrice || product?.price || 0,  // الآن يعرض 18 ج.م
```

### النتيجة
الآن عند فتح نموذج تعديل منتج له باتشات:
- ✅ حقل "سعر البيع" يعرض **سعر الباتش الفعلي** (18 ج.م)
- ✅ الحقل معطّل (disabled) لمنع التعديل المباشر
- ✅ رسالة تنبيه توضح أن السعر المعروض هو سعر الباتش
- ✅ الرسالة تعرض كلاً من سعر الباتش والسعر الأساسي للتوضيح

## مثال عملي

### قبل الإصلاح
```
نموذج تعديل المنتج:
┌─────────────────────────┐
│ سعر البيع: 80 ج.م      │ ← السعر الأساسي (مضلل!)
└─────────────────────────┘

في نقطة البيع:
السعر الفعلي = 18 ج.م  ← من الباتش (مختلف!)
```

### بعد الإصلاح
```
جدول المنتجات:
┌─────────────────────────┐
│ السعر: 18 ج.م          │ ← السعر الفعلي فقط (واضح!)
└─────────────────────────┘

نموذج تعديل المنتج:
┌─────────────────────────────────────────────┐
│ ⚠️ هذا المنتج له دفعات مخزون              │
│                                             │
│ السعر المعروض هو سعر الباتش الحالي (18 ج.م)│
│ السعر الأساسي للمنتج هو 80 ج.م            │
│                                             │
│ سعر البيع: 18 ج.م [معطّل]                 │
└─────────────────────────────────────────────┘

في نقطة البيع:
السعر الفعلي = 18 ج.م  ← متطابق! ✅
```

## فوائد الإصلاح

1. **وضوح للمستخدم:** السعر المعروض = السعر المستخدم في البيع
2. **بساطة العرض:** سعر واحد فقط في جدول المنتجات (لا ارتباك)
3. **اتساق البيانات:** السعر والتكلفة من نفس المصدر (الباتش)
4. **نموذج مبسّط:** حذف حقول الضريبة غير الضرورية
5. **منع الأخطاء:** الحقول معطّلة في النموذج لمنع تعديل البيانات من مكان خاطئ
6. **إرشاد واضح:** رسالة تنبيه في النموذج توجه المستخدم لشاشة إدارة الدفعات
7. **اتساق النظام:** كل الشاشات تستخدم `suggestedPrice` الآن

## كيفية تغيير السعر للمنتجات التي لها باتشات

1. اذهب إلى **إدارة الدفعات** (`/product-batches`)
2. اختر المنتج المطلوب
3. عدّل سعر البيع للباتش المطلوب
4. السعر الجديد سيظهر تلقائياً في:
   - نموذج تعديل المنتج
   - صفحة المنتجات
   - نقطة البيع
   - السلة والفواتير

## ملاحظات مهمة

- **السعر الأساسي** (`price`) لا يزال موجوداً في قاعدة البيانات
- يُستخدم كـ **fallback** عندما لا توجد باتشات للمنتج
- عند إنشاء منتج جديد بدون باتشات، `suggestedPrice = price`
- نظام FEFO يختار الباتش الأقرب للانتهاء تلقائياً
