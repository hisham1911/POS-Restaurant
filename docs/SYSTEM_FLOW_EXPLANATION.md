# 🎯 شرح كامل لنظام KasserPro بعد التحديث

## 📋 الفرق بين الثلاث طرق لإضافة المنتجات

### 1️⃣ ProductFormModal (صفحة المنتجات الرئيسية)
**📍 المكان:** `/products` → زر "إضافة منتج"

**🎯 الاستخدام:**
- إضافة منتجات **دائمة** في الكتالوج
- تفاصيل كاملة: صور، تكلفة، مخزون، أسعار فروع
- للمنتجات التي تبيعها **بشكل متكرر**

**⚙️ الخيارات:**
```
نوع المنتج:
├─ منتج مادي (Physical)
│  ├─ يتم تتبع المخزون تلقائياً
│  ├─ ينقص المخزون عند البيع
│  └─ يزيد المخزون عند المرتجعات
│
└─ خدمة (Service)
   ├─ بدون تتبع مخزون
   ├─ مثال: "خدمة توصيل"، "صيانة"
   └─ لا يؤثر على المخزون
```

**📝 مثال:**
- منتج مادي: "قهوة تركي" - 25 جنيه - مخزون 100 كيس
- خدمة: "خدمة توصيل" - 15 جنيه - بدون مخزون

---

### 2️⃣ ProductQuickCreateModal (في POS)
**📍 المكان:** `/pos` → زر "منتج جديد" (أزرق)

**🎯 الاستخدام:**
- إضافة منتج **سريع** أثناء البيع
- يُحفظ في الكتالوج ويظهر للطلبات المستقبلية
- للمنتجات الجديدة التي ستبيعها **مرات متعددة**

**⚙️ الخيارات:**
```
نوع المنتج:
├─ منتج مادي → يتتبع المخزون
└─ خدمة → بدون مخزون
```

**📝 مثال:**
- الكاشير يبيع منتج جديد "عصير برتقال طازج"
- يضيفه سريعاً بالسعر والنوع
- المنتج يُحفظ ويظهر في الكتالوج للمرات القادمة

---

### 3️⃣ CustomItemModal (في POS)
**📍 المكان:** `/pos` → زر "منتج مخصص" (برتقالي/ثانوي)

**🎯 الاستخدام:**
- إضافة منتج **مؤقت** للطلب الحالي فقط
- **لا يُحفظ** في الكتالوج
- للمنتجات **غير المتكررة** أو الحالات الخاصة

**⚙️ الخيارات:**
```
- الاسم
- السعر
- الكمية
- معدل الضريبة (اختياري)
- ملاحظات
```

**📝 أمثلة:**
- "خدمة تغليف هدية خاصة" - 10 جنيه - مرة واحدة
- "رسوم إضافية" - 5 جنيه
- "خصم خاص للعميل" - -20 جنيه (سعر سالب)
- "منتج تجريبي" - 50 جنيه

**⚠️ مهم:**
- المنتج المخصص **لا يظهر** في قائمة المنتجات
- **لا يؤثر** على المخزون أبداً
- يظهر فقط في الفاتورة الحالية

## 🔄 سير العمل الكامل

### 📦 سيناريو 1: بيع منتج مادي عادي
```
1. الكاشير يفتح وردية
2. يختار "قهوة تركي" من المنتجات
3. يضيفها للسلة (كمية 2)
4. يضغط "إتمام الدفع"
5. النظام:
   ✅ يتحقق من المخزون (هل يوجد 2 كيس؟)
   ✅ يحسب السعر + الضريبة
   ✅ يسجل الطلب
   ✅ ينقص المخزون (-2)
   ✅ يسجل حركة المخزون
```

### 🛠️ سيناريو 2: بيع خدمة
```
1. الكاشير يختار "خدمة توصيل" (Service)
2. يضيفها للسلة
3. يضغط "إتمام الدفع"
4. النظام:
   ✅ يحسب السعر + الضريبة
   ✅ يسجل الطلب
   ⏭️ يتخطى المخزون (لأنها خدمة)
```

### ✨ سيناريو 3: بيع منتج مخصص
```
1. الكاشير يضغط "منتج مخصص"
2. يكتب: "تغليف هدية خاصة" - 10 جنيه
3. يضيفها للسلة
4. يضغط "إتمام الدفع"
5. النظام:
   ✅ يحسب السعر + الضريبة
   ✅ يسجل الطلب
   ⏭️ يتخطى المخزون (لأنه custom item)
   ⏭️ لا يحفظ المنتج في الكتالوج
```

### 🔄 سيناريو 4: مرتجع منتج مادي
```
1. العميل يرجع "قهوة تركي" (كمية 1)
2. الكاشير يضغط "مرتجع"
3. النظام:
   ✅ ينشئ طلب مرتجع (Return Order)
   ✅ يرجع المبلغ للعميل
   ✅ يزيد المخزون (+1)
   ✅ يسجل حركة المخزون
```

### 🔄 سيناريو 5: مرتجع خدمة
```
1. العميل يرجع "خدمة توصيل"
2. الكاشير يضغط "مرتجع"
3. النظام:
   ✅ ينشئ طلب مرتجع
   ✅ يرجع المبلغ للعميل
   ⏭️ لا يؤثر على المخزون (لأنها خدمة)
```

### 🔄 سيناريو 6: مرتجع منتج مخصص
```
1. العميل يرجع "تغليف هدية خاصة"
2. الكاشير يضغط "مرتجع"
3. النظام:
   ✅ ينشئ طلب مرتجع
   ✅ يرجع المبلغ للعميل
   ⏭️ لا يؤثر على المخزون (لأنه custom item)
```

## 🗄️ البيانات في قاعدة البيانات

### Products Table
```sql
Id | Name              | Type     | TrackInventory | StockQuantity
---|-------------------|----------|----------------|---------------
1  | قهوة تركي         | Physical | true           | 100
2  | خدمة توصيل        | Service  | false          | NULL
3  | شاي بالنعناع     | Physical | true           | 50
```

### Orders Table
```sql
Id | OrderNumber | Status    | Total
---|-------------|-----------|-------
1  | ORD-001     | Completed | 65.00
2  | RET-001     | Completed | -25.00 (مرتجع)
```

### OrderItems Table
```sql
Id | OrderId | ProductId | IsCustomItem | CustomName              | Quantity | Total
---|---------|-----------|--------------|-------------------------|----------|-------
1  | 1       | 1         | false        | NULL                    | 2        | 50.00
2  | 1       | 2         | false        | NULL                    | 1        | 15.00
3  | 1       | NULL      | true         | "تغليف هدية خاصة"      | 1        | 10.00
4  | 2       | 1         | false        | NULL                    | 1        | -25.00
```

**📝 ملاحظات:**
- `ProductId = NULL` + `IsCustomItem = true` → منتج مخصص
- `ProductId != NULL` + `IsCustomItem = false` → منتج من الكتالوج
- المرتجعات: `Quantity` و `Total` سالبة

## ✅ الخلاصة

### متى تستخدم كل طريقة؟

| الطريقة | متى تستخدمها | يُحفظ في الكتالوج | يؤثر على المخزون |
|---------|--------------|-------------------|-------------------|
| **ProductFormModal** | منتجات دائمة بتفاصيل كاملة | ✅ نعم | ✅ نعم (إذا Physical) |
| **ProductQuickCreateModal** | منتج جديد سريع أثناء البيع | ✅ نعم | ✅ نعم (إذا Physical) |
| **CustomItemModal** | منتج مؤقت للطلب الحالي فقط | ❌ لا | ❌ لا |

### القواعد الذهبية:

1. **منتج مادي (Physical)** = يتتبع المخزون تلقائياً
2. **خدمة (Service)** = بدون مخزون
3. **منتج مخصص (Custom Item)** = لا يُحفظ ولا يؤثر على المخزون
4. **المرتجعات** = تعكس كل شيء (مخزون + مبالغ)

### الأمان:

- ✅ التحقق من المخزون **قبل** إتمام الطلب
- ✅ التحقق من المخزون **أثناء** إتمام الطلب (داخل Transaction)
- ✅ إذا تغير المخزون بين الخطوتين → رفض الطلب
- ✅ كل العمليات داخل Transaction (إما تنجح كلها أو تفشل كلها)
