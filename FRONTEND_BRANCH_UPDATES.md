## 🎯 السلوك المتوقع

### عند إضافة منتج جديد:

#### من صفحة المنتجات الرئيسية:
1. **بدون تحديد كميات مخصصة** (الافتراضي):
   - الكمية المدخلة → الفرع الحالي
   - الفروع الأخرى → صفر

2. **مع تحديد كميات مخصصة**:
   - يظهر قائمة بكل الفروع
   - يمكن تحديد كمية مختلفة لكل فرع
   - مفيد عند استلام شحنة موزعة على عدة فروع

#### من POS (Quick Create):
- الكمية المدخلة → الفرع الحالي فقط
- الفروع الأخرى → صفر
- مناسب للإضافة السريعة أثناء البيع

#### من فواتير الشراء (Quick Add):
- المنتج يُضاف بكمية صفر للفرع الحالي
- الكمية الفعلية تُضاف عند تأكيد فاتورة الشراء
- منطقي لأن الكمية ستأتي من الفاتورة نفسها

## 📊 أمثلة عملية

### مثال 1: إضافة منتج من صفحة المنتجات
```
المستخدم: في الفرع الرئيسي
المنتج: قهوة تركي
الكمية المدخلة: 100

النتيجة:
- الفرع الرئيسي: 100 ✓
- فرع المعادي: 0
- فرع المهندسين: 0
```

### مثال 2: إضافة منتج بكميات مخصصة
```
المستخدم: في الفرع الرئيسي
المنتج: شاي أخضر
☑ تحديد كمية مخصصة لكل فرع

الكميات المدخلة:
- الفرع الرئيسي: 50
- فرع المعادي: 30
- فرع المهندسين: 20

النتيجة:
- الفرع الرئيسي: 50 ✓
- فرع المعادي: 30 ✓
- فرع المهندسين: 20 ✓
```

### مثال 3: إضافة منتج من POS
```
المستخدم: في فرع المعادي
المنتج: كابتشينو
الكمية: 25

النتيجة:
- الفرع الرئيسي: 0
- فرع المعادي: 25 ✓
- فرع المهندسين: 0
```

### مثال 4: إضافة منتج من فاتورة شراء
```
المستخدم: في الفرع الرئيسي
المنتج الجديد: سكر أبيض
(يُضاف بكمية 0 أولاً)

ثم في الفاتورة:
- المنتج: سكر أبيض
- الكمية: 100 كيس
- عند التأكيد → الفرع الرئيسي: 100 ✓
```

## 🔄 نقل المخزون بين الفروع

لنقل كمية من فرع لآخر، استخدم:
**Inventory → Transfers**

```
مثال:
من: الفرع الرئيسي (100 وحدة)
إلى: فرع المعادي (0 وحدة)
الكمية: 30

بعد التحويل:
- الفرع الرئيسي: 70
- فرع المعادي: 30
```

## 💡 نصائح للمستخدمين

### متى تستخدم الكميات المخصصة؟
✅ **استخدمها عند**:
- استلام شحنة موزعة على عدة فروع
- جرد مخزون شامل لكل الفروع
- إعداد مخزون أولي لنظام جديد

❌ **لا تستخدمها عند**:
- إضافة منتج عادي (استخدم الافتراضي)
- إضافة سريعة من POS
- إضافة من فاتورة شراء

### كيف أضيف كمية لفرع آخر؟
1. **الطريقة الأولى**: استخدم Inventory Transfer
2. **الطريقة الثانية**: أنشئ فاتورة شراء في الفرع المستهدف
3. **الطريقة الثالثة**: استخدم Inventory Adjustment (Admin فقط)
