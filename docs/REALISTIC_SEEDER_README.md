# 🚀 RealisticDataSeeder - دليل سريع

## ماذا يفعل؟

يولد **120 يوم** من البيانات الواقعية لسوبر ماركت الخير، يشمل:

- � **121 وردية** (وردية واحدة يومياً × 121 يوم)
- 🛒 **~1,500 طلب** (10-25 طلب/يوم حسب نوع اليوم)
- � **طرق دفع متنوعة**: Cash (50%)، Card (35%)، Fawry (10%)، Bank Transfer (5%)
- � **حالات متنوعة**: Completed (85%)، Cancelled (10%)، Refunded (5%)
- � **20% من الطلبات** بها ديون (AmountPaid < Total)

---

## كيف تستخدمه؟

### الطريقة الأولى: تلقائي (موصى به)

```bash
cd backend/KasserPro.API
dotnet run
```

سيعمل تلقائياً عند أول تشغيل!

### الطريقة الثانية: إعادة تحميل

```bash
# احذف قاعدة البيانات
rm backend/KasserPro.API/kasserpro.db

# شغل التطبيق
cd backend/KasserPro.API
dotnet run
```

## ملاحظات مهمة

1. ⚠️ **يتحقق من وجود البيانات** - إذا كان عدد الطلبات > 500 يتخطى التحميل
2. ⚠️ **وقت التنفيذ**: ~10-20 ثانية
3. ⚠️ **نسخة مبسطة** - يركز على الورديات والطلبات فقط

---

## السيناريوهات المغطاة

### ✅ الطلبات
- 85% مكتملة
- 10% ملغاة
- 5% مرتجعة
- 20% بها ديون

### ✅ طرق الدفع
- 50% Cash
- 35% Card
- 10% Fawry
- 5% Bank Transfer

### ✅ فواتير الشراء
- 60% مدفوعة بالكامل
- 30% مدفوعة جزئياً
- 10% آجلة (ديون موردين)

### ✅ المصروفات
- رواتب شهرية (18,000 ج.م)
- إيجار شهري (12,000 ج.م)
- فواتير خدمات (كهرباء، مياه)
- مصروفات عشوائية (صيانة، مواصلات، تسويق)

## ⚠️ ملاحظة

هذه نسخة **مبسطة** من الـ Seeder تركز على الورديات والطلبات فقط.
للحصول على بيانات أكثر شمولاً (موردين، فواتير شراء، مصروفات، إلخ)، 
يمكن توسيع الكود لاحقاً.
